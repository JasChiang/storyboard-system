# 🔍 如何確定 Gemini 有看到影片

本指南說明如何驗證 Gemini AI 是否真的分析了影片內容，而不是只根據文字描述猜測。

---

## ✅ 已實施的驗證機制

### 1. 視覺確認欄位（推薦）⭐

#### 工作原理

系統已修改提示詞，**強制要求** Gemini 在每個場景建議中描述實際看到的內容：

**提示詞要求**：
```
⚠️ **重要**：你必須在每個場景建議中包含 `visualConfirmation` 欄位，
簡短描述你在該影片中實際看到的內容（1-2 句話），以確認你真的觀看了影片。
```

**JSON 輸出格式**：
```json
{
  "sceneId": "scene-123",
  "visualConfirmation": "我看到產品在黑色背景中緩慢旋轉，前半秒靜止，後段光線漸暗",
  "inPoint": 0.5,
  "outPoint": 4.8,
  "transition": "crossfade",
  "effects": ["color_correction", "glow"]
}
```

#### UI 顯示位置

在「Blender 匯出」頁面 → 影片分析結果 → 場景剪輯建議區塊：

```
場景: scene-123
✓ AI 確認: 我看到產品在黑色背景中緩慢旋轉，前半秒靜止，後段光線漸暗
入點: 0.5s | 出點: 4.8s
轉場: crossfade
```

#### 判斷標準

| 情況 | 描述 | 結論 |
|------|------|------|
| ✅ **有看到影片** | 描述具體的視覺細節（動作、光線、顏色、時間點） | Gemini 真的分析了影片 |
| ⚠️ **可能沒看到** | 只是重複分鏡腳本的文字 | 可能只根據文字猜測 |
| ❌ **肯定沒看到** | 內容與實際影片不符 | 影片未被成功傳遞 |

**範例對比**：

✅ **好的確認**（基於視覺）：
- "產品從左側緩緩移入畫面，旋轉過程中反光明顯，結尾處光線變暗"
- "開場前0.3秒畫面靜止，中段旋轉速度加快，電池指示燈持續亮起"

❌ **不好的確認**（只是重述文字）：
- "MEGA KING行動電源緩緩旋轉，展現高級質感"（← 這只是複製分鏡描述）
- "產品旋轉展示"（← 太籠統）

## 🔧 故障排除

### 問題 1：所有場景都沒有 `visualConfirmation`

**可能原因**：
1. Gemini 無法訪問影片 URL（網絡問題、URL 過期）
2. Gemini API 配額不足，無法處理影片
3. 影片格式不支援

**解決方案**：
1. 檢查終端日誌中的 URI 是否可訪問
2. 在瀏覽器中手動打開影片 URL 測試
3. 切換到 `gemini-1.5-pro` 模型（更高配額）
4. 確認 Fal.ai 影片 URL 有效期

---

### 問題 2：`visualConfirmation` 描述不準確

**可能原因**：
1. Gemini "幻覺"（AI 猜測而非真實觀看）
2. 影片內容與預期不符

**驗證方法**：
1. 手動觀看影片，對比 AI 描述
2. 檢查 AI 描述的具體細節（時間點、動作、光線）
3. 如果多個場景都不準確 → 可能沒真正觀看

---

### 問題 3：終端顯示「包含視覺確認: 否」

**立即行動**：
1. 檢查 Gemini API Key 是否有效
2. 檢查 `.env.local` 的 `GEMINI_MODEL` 設定
3. 確認影片 URL 未過期
4. 查看終端是否有錯誤訊息

---

## 📋 標準驗證流程

### 步驟 1：檢查終端日誌
```bash
# 應該看到：
🎬 Gemini 影片分析請求:
  - 影片數量: 3  ← 確認有影片
  - 場景數量: 7

✅ Gemini 分析完成:
  - 包含視覺確認: 是 ✓  ← 關鍵指標
```

### 步驟 2：檢查 UI 顯示
- 展開「場景剪輯建議」
- 查看是否有綠色的 `✓ AI 確認` 標籤
- 閱讀確認內容是否具體

### 步驟 3：手動驗證
1. 打開實際影片
2. 對比 AI 描述的細節：
   - 時間點是否準確？
   - 動作描述是否吻合？
   - 光線、顏色描述是否正確？

### 步驟 4：判定結論

| 檢查項 | 結果 | 結論 |
|--------|------|------|
| 終端日誌 | ✅ 包含視覺確認: 是 | |
| UI 顯示 | ✅ 有綠色確認標籤 | |
| 內容準確性 | ✅ 描述與影片吻合 | **Gemini 真的看到了影片** ✅ |
| | | |
| 終端日誌 | ⚠️ 包含視覺確認: 否 | |
| UI 顯示 | ❌ 沒有確認標籤 | |
| 內容準確性 | ❌ 只重述文字/不準確 | **Gemini 沒有看到影片** ❌ |

---

## 🎯 總結

**最可靠的驗證方法**：

1. ⭐ **主要依據**：UI 中的 `✓ AI 確認` 內容是否具體且準確
2. 📊 **輔助依據**：終端日誌顯示「包含視覺確認: 是 ✓」
3. 🔍 **終極驗證**：手動對比影片內容與 AI 描述

**如果確認 Gemini 沒看到影片**：
- 檢查影片 URL 是否公開可訪問
- 考慮改用 Gemini Files API 上傳影片（更穩定）
- 檢查 API 配額和網絡連接
