# 🚀 快速開始指南

## ✅ 已完成的功能

### 1️⃣ 全局角色庫系統
**頁面**: `/characters`

**功能**:
- ✅ 建立和管理可重複使用的 IP 角色
- ✅ 多視角圖片上傳（正面、側面、3/4侧等）
- ✅ AI 自動描述角色特徵
- ✅ 標籤搜尋和類型篩選
- ✅ 使用次數統計

**適用場景**: 公司吉祥物、長期使用的角色、多專案重複使用

### 2️⃣ 角色選擇與整合
**頁面**: `/project/[id]/storyboard`

**功能**:
- ✅ 从角色庫快速選擇角色
- ✅ 支援暫時上傳 + 角色庫混合使用
- ✅ AI 生成劇本时自動使用 `<角色名>` 標記
- ✅ 實时顯示已选角色列表

**工作流程**:
```
方式A：角色庫選擇
┌─────────────┐
│ 點擊「從角色庫選擇」│
└──────┬──────┘
       ↓
┌─────────────┐
│ 選擇角色和視角  │
└──────┬──────┘
       ↓
┌─────────────┐
│ 自動加入參考圖  │
└─────────────┘

方式B：暫時上傳（保留）
┌─────────────┐
│  直接上傳圖片   │
└──────┬──────┘
       ↓
┌─────────────┐
│ AI 自動描述   │
└──────┬──────┘
       ↓
┌─────────────┐
│ 可選：加入角色庫│
└─────────────┘
```

### 3️⃣ Remotion 本地渲染
**頁面**: `/project/[id]/export`

**功能**:
- ✅ 一鍵自動合成影片
- ✅ 自動加入轉場效果（Dissolve/Fade）
- ✅ 字幕底部顯示
- ✅ Ken Burns 效果（圖片慢慢放大）
- ✅ 本地渲染，完全免費
- ✅ 實时進度顯示
- ✅ 線上預覽和下載

**技術棧**:
- React + TypeScript（統一技術棧）
- `@remotion/renderer`（本地渲染引擎）
- H.264 編碼，1080p 輸出

### 4️⃣ 双模式匯出
**頁面**: `/project/[id]/export`

**两种選擇**:

#### 模式 A：Remotion 快速渲染（推薦）
- ⚡ 一鍵完成，無需手動操作
- 🎬 自動轉場和字幕
- 💰 免費，本地渲染
- 適合：快速出片、標準流程

#### 模式 B：Blender 專業匯出
- 🎨 完全控制，手動精修
- 🔧 複雜特效（Compositor）
- 🎯 高級調色
- 適合：專業制作、複雜需求

## 📦 安裝與啟動

### 1. 安裝依赖

```bash
npm install
```

新增依赖包：
- `remotion` - 影片合成庫
- `@remotion/cli` - 命令行工具
- `@remotion/bundler` - 打包工具
- `@remotion/renderer` - 本地渲染引擎

### 2. 啟動開發伺服器

```bash
npm run dev
```

造訪 http://localhost:3000

### 3. 可選：開啟 Remotion Studio

```bash
npm run remotion:studio
```

視覺化預覽和除錯 Remotion 影片组件。

## 🎯 完整工作流程

### 場景 A：使用既有 IP 角色（推薦）

```
第一次使用：
1. 造訪「角色庫」(/characters)
2. 點擊「新增角色」
3. 上傳多個視角的圖片（正面、側面等）
4. AI 自動描述或手動輸入
5. 加入標籤（如「可爱」「卡通」）
6. 保存到角色庫

後續使用：
1. 建立新專案
2. 在分鏡腳本頁面點擊「從角色庫選擇」
3. 選擇角色和視角 → 自動引用
4. 輸入劇本 → AI 會使用 <角色名> 標記
5. 生成圖片/影片
6. 匯出 → 選擇「Remotion 快速渲染」
7. 下載成品影片 🎉
```

### 場景 B：暫時角色（一次性專案）

```
1. 建立專案
2. 在分鏡腳本頁面直接上傳參考圖
3. AI 自動描述
4. 輸入劇本並生成
5. 生成圖片/影片
6. 可選：將暫時角色「加入角色庫」（待實作）
7. 匯出渲染
```

## 🔧 設定說明

### API 金鑰
在分鏡腳本頁面點擊「API 設定」或設定環境變量：

```bash
# .env.local
OPENROUTER_API_KEY=sk-or-v1-...    # 劇本生成
FAL_KEY=...                         # 圖片/影片生成
GEMINI_API_KEY=...                  # 影片分析（Blender 模式）
```

### Remotion 設定
編輯 `remotion.config.ts`:

```typescript
Config.setVideoImageFormat('jpeg'); // 或 'png'
Config.setCodec('h264');            // 或 'h265', 'vp8'
Config.setOutputLocation('out/video.mp4');
```

## 🐛 已知限制

1. **Remotion 渲染时间**: 首次渲染需要打包，約 30-60 秒（後續更快）
2. **渲染進度**: 目前是模拟進度，實际應从後端取得
3. **影片編碼**: 僅支援 H.264，不支援 HDR
4. **最大時長**: 建議单個影片不超过 5 分鐘
5. **記憶體占用**: 多場景影片可能占用较多記憶體

## 🎬 範例使用

### 建立第一個角色

```bash
# 1. 造訪角色庫
http://localhost:3000/characters

# 2. 點擊「新增角色」
# 3. 填写資訊：
名稱: 小熊吉祥物
類型: 角色
標籤: 可爱, 卡通, 公司IP

# 4. 上傳視角：
正面: bear-front.png
側面: bear-side.png
3/4侧: bear-three-quarter.png

# 5. 點擊「建立角色」✅
```

### 使用角色建立專案

```bash
# 1. 首頁點擊「新專案」
# 2. 前往分鏡腳本頁面
# 3. 點擊「從角色庫選擇」
# 4. 選擇「小熊吉祥物」→ 正面視角
# 5. 輸入劇本：
"小熊在公园裡散步，遇到了小兔子。
他们一起玩耍，度过了愉快的一天。"

# 6. 生成分鏡 → 生成圖片 → 生成影片
# 7. 匯出 → Remotion 快速渲染 → 下載成品 🎉
```

## ⚡ 性能優化建議

### 角色庫
- 圖片建議大小：< 2MB
- 推薦格式：JPEG（文件小）或 PNG（透明背景）
- 視角数量：3-6 個（不要太多）

### Remotion 渲染
- 場景数量：建議 < 20 個
- 单場景時長：2-10 秒
- 圖片分辨率：1920x1080（不要用 4K 原圖）
- 影片格式：MP4 (H.264)
