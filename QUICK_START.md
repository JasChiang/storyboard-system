# 🚀 快速开始指南

## ✅ 已完成的功能

### 1️⃣ 全局角色库系统
**页面**: `/characters`

**功能**:
- ✅ 创建和管理可复用的 IP 角色
- ✅ 多视角图片上传（正面、侧面、3/4侧等）
- ✅ AI 自动描述角色特征
- ✅ 标签搜索和类型筛选
- ✅ 使用次数统计

**适用场景**: 公司吉祥物、长期使用的角色、多项目复用

### 2️⃣ 角色选择与集成
**页面**: `/project/[id]/storyboard`

**功能**:
- ✅ 从角色库快速选择角色
- ✅ 支持临时上传 + 角色库混合使用
- ✅ AI 生成剧本时自动使用 `<角色名>` 标记
- ✅ 实时显示已选角色列表

**工作流程**:
```
方式A：角色库选择
┌─────────────┐
│ 点击「从角色库选择」│
└──────┬──────┘
       ↓
┌─────────────┐
│ 选择角色和视角  │
└──────┬──────┘
       ↓
┌─────────────┐
│ 自动加入参考图  │
└─────────────┘

方式B：临时上传（保留）
┌─────────────┐
│  直接上传图片   │
└──────┬──────┘
       ↓
┌─────────────┐
│ AI 自动描述   │
└──────┬──────┘
       ↓
┌─────────────┐
│ 可选：加入角色库│
└─────────────┘
```

### 3️⃣ Remotion 本地渲染
**页面**: `/project/[id]/export`

**功能**:
- ✅ 一键自动合成视频
- ✅ 自动添加转场效果（Dissolve/Fade）
- ✅ 字幕底部显示
- ✅ Ken Burns 效果（图片慢慢放大）
- ✅ 本地渲染，完全免费
- ✅ 实时进度显示
- ✅ 在线预览和下载

**技术栈**:
- React + TypeScript（统一技术栈）
- `@remotion/renderer`（本地渲染引擎）
- H.264 编码，1080p 输出

### 4️⃣ 双模式导出
**页面**: `/project/[id]/export`

**两种选择**:

#### 模式 A：Remotion 快速渲染（推荐）
- ⚡ 一键完成，无需手动操作
- 🎬 自动转场和字幕
- 💰 免费，本地渲染
- 适合：快速出片、标准流程

#### 模式 B：Blender 专业导出
- 🎨 完全控制，手动精修
- 🔧 复杂特效（Compositor）
- 🎯 高级调色
- 适合：专业制作、复杂需求

## 📦 安装与启动

### 1. 安装依赖

```bash
npm install
```

新增依赖包：
- `remotion` - 视频合成库
- `@remotion/cli` - 命令行工具
- `@remotion/bundler` - 打包工具
- `@remotion/renderer` - 本地渲染引擎

### 2. 启动开发服务器

```bash
npm run dev
```

访问 http://localhost:3000

### 3. 可选：打开 Remotion Studio

```bash
npm run remotion:studio
```

可视化预览和调试 Remotion 视频组件。

## 🎯 完整工作流程

### 场景 A：使用既有 IP 角色（推荐）

```
第一次使用：
1. 访问「角色库」(/characters)
2. 点击「新增角色」
3. 上传多个视角的图片（正面、侧面等）
4. AI 自动描述或手动输入
5. 添加标签（如「可爱」「卡通」）
6. 保存到角色库

后续使用：
1. 创建新项目
2. 在分镜脚本页面点击「从角色库选择」
3. 选择角色和视角 → 自动引用
4. 输入剧本 → AI 会使用 <角色名> 标记
5. 生成图片/视频
6. 导出 → 选择「Remotion 快速渲染」
7. 下载成品视频 🎉
```

### 场景 B：临时角色（一次性项目）

```
1. 创建项目
2. 在分镜脚本页面直接上传参考图
3. AI 自动描述
4. 输入剧本并生成
5. 生成图片/视频
6. 可选：将临时角色「加入角色库」（待实现）
7. 导出渲染
```

## 🔧 配置说明

### API 金鑰
在分镜脚本页面点击「API 设定」或设置环境变量：

```bash
# .env.local
OPENROUTER_API_KEY=sk-or-v1-...    # 剧本生成
FAL_KEY=...                         # 图片/视频生成
GEMINI_API_KEY=...                  # 视频分析（Blender 模式）
```

### Remotion 配置
编辑 `remotion.config.ts`:

```typescript
Config.setVideoImageFormat('jpeg'); // 或 'png'
Config.setCodec('h264');            // 或 'h265', 'vp8'
Config.setOutputLocation('out/video.mp4');
```

## 🐛 已知限制

1. **Remotion 渲染时间**: 首次渲染需要打包，约 30-60 秒（后续更快）
2. **渲染进度**: 当前是模拟进度，实际应从后端获取
3. **视频编码**: 仅支持 H.264，不支持 HDR
4. **最大时长**: 建议单个视频不超过 5 分钟
5. **内存占用**: 多场景视频可能占用较多内存

## 🎬 示例使用

```bash
# 1. 访问角色库
```

## ⚡ 性能优化建议

### 角色库
- 图片建议大小：< 2MB
- 推荐格式：JPEG（文件小）或 PNG（透明背景）
- 视角数量：3-6 个（不要太多）

### Remotion 渲染
- 场景数量：建议 < 20 个
- 单场景时长：2-10 秒
- 图片分辨率：1920x1080（不要用 4K 原图）
- 视频格式：MP4 (H.264)
